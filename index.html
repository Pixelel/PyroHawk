The Problem

Prescribed burning works. Studies show it reduces burn severity by around 16% and cuts emissions by around 14%. The issue is that doing it requires crews to go into rough, hard to reach terrain, which makes the process slow and expensive.

PyroHawk is my attempt to fix that. Instead of sending people in, the drone flies to the target area and drops ignited pellets at specific GPS coordinates. The whole thing runs autonomously, no one has to enter the burn zone.

The image above shows a similar existing concept, a heavy lift drone carrying a pellet dispenser underneath. PyroHawk builds on this with a custom dropper mechanism and a full AI navigation stack.

System Overview

How It Works

01
Airframe
Built on an Echo 5" frame with four Readytosky RS2205 motors and HQProp 3-blade propellers. The SpeedyBee V5 flight stack sits on vibration damping pads at the center. A GPS module feeds position data to the flight controller.
02
Pellet Dropper
A conical hopper feeds pellets one at a time through a servo controlled gate. Each pellet goes down a ceramic tube, gets clamped, lit by a nichrome wire ignition system, then dropped at the target. Everything in the drop path is fire resistant.
03
AI Navigation
A Swin-UNet vision model reads the camera feed and maps the terrain. A modified TrailNet model takes that map and picks a target coordinate. ArduPilot then flies the drone there using MAVLink in Guided Flight Mode. The loop repeats automatically.
04
Onboard Computer
A Jetson Orin-Nano handles all the AI processing on board in real time. It runs the terrain model, the navigation model, and communicates with the flight controller at the same time, all without any ground input.
05
Ignition Safety
The ignition system uses Arduino controlled relays to fire nichrome wire for a set duration, then cut power before the pellet drops. Ignition is tested completely separately from the drone in a safe environment, never during flight tests.
06
CAD Design
The full mechanical layout is being designed in Fusion 360 before any parts are built. This includes the hopper, servo mount, release gate, and bracket placements. The 3D printed parts use PA6-CF filament for heat resistance.
Payload Detail

Pellet Drop Mechanism

Pellet flow — side cross section
1
Hopper Storage
Wood pellets sit in the conical hopper and fall by gravity toward the exit at the bottom. The shape keeps them in a single file line so they feed consistently.
2
Servo Gate
An MG996R servo pushes one pellet at a time into the drop tube using a small lever, then resets to block the next one. This makes sure only one pellet drops per cycle.
3
Ceramic Drop Tube
The pellet slides down a ceramic tube into a clamp at the bottom. Ceramic is used here because it can handle the heat from ignition without getting damaged.
4
Nichrome Ignition
While the pellet is held in the clamp, an Arduino controlled relay sends power to nichrome wire wrapped around it. The wire heats up and lights the pellet. The relay cuts power before the clamp opens.
5
Drop
The clamp releases and the pellet falls to the GPS target below. ArduPilot records the drop and the AI model picks the next location. The cycle repeats until the mission is done.

Development Plan

Build Roadmap

Mechanical
1
CAD in Fusion 360
Model the full mechanical layout including the hopper, servo mount, release gate, and wiring paths. Lets us check spacing and balance before ordering parts.
2
Drone Assembly and Testing
Put together the airframe with motors, flight stack, GPS, and battery. Test that it flies stable and can handle the weight of the full payload.
3
Build the Pellet Dropper
3D print the hopper and drop tube in PA6-CF filament. Attach the servo gate and run tests to confirm single pellet release works correctly.
4
Full Assembly
Mount the dropper onto the drone frame and make sure everything fits. No rotor interference. Drone is ready for software testing at this point.
Software
1
Modify TrailNet
Add a depth estimation layer and new output heads for Gap Center, Object Proximity, and a danger score. Get it ready to train on the FinnWoodlands Dataset.
2
ArduPilot Setup and TrailNet Training
Set up MAVLink and Guided Flight Mode in ArduPilot. Train the TrailNet model and run accuracy tests. Finalize the weights.
3
Train Swin-UNet and Connect Pipeline
Train the Swin-UNet model on its own, then connect it to TrailNet. Run the full AI pipeline on the Jetson Orin-Nano and link it to ArduPilot.
4
Full Integration Testing
Run the whole autonomous loop on the finished drone. Test vision to navigation to drop from start to finish. Fix any issues. No ignition during these tests.
